[Html/CSS Common Rule]

1. HTML
	- 앞은 탭(1탭=4스페이스)으로 들여쓰고 그 이외 부분은 1스페이스로 합니다.
	- 불필요한 공백 제거 및 줄 끝 공백 제거가 되어야 합니다.
	- HTML 내부의 태그 및 속성은 모두 소문자 표기를 원칙으로 합니다. 속성값은 쌍 따옴표("")를 묶어 줍니다.
	- xhtml 문법으로 작업 ex) 

	- [html 파일 네이밍] : 화면ID.html

2. CSS 작성 순서
		- class는 탭(1탭=4스페이스)으로 들여쓰고 속성 사이는 1스페이스로 합니다.
		- CSS3의 자식선택자, 인접형제선택자, A태그의 이외의 의사클래스 사용을 하지 않는다.
		- 변경되는 범위가 큰 순서부터 작성되도록 합니다.
			display / overflow / visibility / position(left, top) / float(clear) /
			width / height
			margin / padding /
			background / border
			text-align(vertical-align) / color / font(weight, size, line-height) / other
			z-index

3. Image
	- 유형별로 하나의 이미지로 합치고, DOM 구조는 아이콘 부분을  사용하여 CSS에서 이미지 sprite 방법을 기본으로 한다.
	- 테이블 위의 아이콘에 대한 설명이 있고, 테이블내에 아이콘으로만 정보를 제공할 경우는 img 태그 사용하며 alt 에는 "무슨무슨 아이콘" 으로 대체텍스트를 적용
		ex) 
	- 블릿이미지나 이미지 sprite 방법을 사용 시 DOM 구조와 CSS 속성이 복잡한 케이스일 경우는  태그 및 이미지 sprite 방법을 사용하지 않고 배경이미지로 사용한다.

4. Name Guide
	- 파일명 & Class 명은 카멜 표기법으로 표기합니다. (첫 음절은 소문자 다음음절의 시작단어는 대문자)
		예) btn + area => btnArea
	- 이미지 명은 모두 소문자로 표기합니다. 단어 구분자는 " _ "로 통일합니다. ( "_" 외 다른 특수문자 사용 금지 )
		예) btn + area => btn_area.gif
	- 의미를 살릴 수 있는 단어의 조합으로 생성합니다.
	- 동일 형태의 이미지, class 발생 시 순번 표기합니다.
		이미지 예) cont_tit01.gif, cont_tit02.gif
		Class 예) contTit01, contTit02

5. Mail Form Guide
	- 메일 인코딩은 euc-kr을 원칙으로 합니다.
	- 링크는 새창으로 제작합니다.   예) 
	- 이미지 경로는 절대 경로로 제작합니다.   예) http://www.ahnlab.com/...
	- 메일폼 제작 후 아웃룩, 포털사이트(naver, nate, hanmail, gmail, hotmail 등)에서 메일테스트 진행 부탁드립니다.

[개발자 사항]

1. html 파일 관리
	- 작업된 html 파일은 SVN으로 관리되며 퍼블리셔 개별적으로 로컬 작업 후 SVN 서버로 commit 됩니다.
	- html 파일만 관리되며 디자인 파일 및 기획 문서는 해당되지 않습니다.

2. html 작업 상태 확인
	- 좌측 트리메뉴에서 각 html 작업 현황이 색상으로 표시되며, 개발에서는 최종 검수가 끝나고 검정색으로 마킹된 파일만 체크하셔서 작업을 시작합니다.
	- 검수 완료 후 개발 전달 된 파일이 재수정이 되어 업데이트 되는 경우에는 붉은색으로 해당파일이 나타나며, 수정 일자가 추가됩니다
		ex) 140204 수정
	- 개발전달 후 수정되어 업데이트 된 파일내에는 오류방지와 편의를 위해 수정부분에 대한 주석이 추가됩니다. 작업시 참고하시면 됩니다.
