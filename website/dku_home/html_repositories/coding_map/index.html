개발자 알림
  • 완료일을 클릭하시면 html 페이지를 보실 수 있습니다.

  • 내부 링크일 경우 수정 이력은 원본 파일에만 표기합니다.
개발자 웹 접근성 유의사항 보기
  • 기본언어 표시
    iframe에 표시되는 페이지를 포함하여 모든 웹페이지에는 기본언어를 표시해야합니다.
    국문 : lang="ko" xml:lang="ko"
    영문 : lang="en" xml:lang="en"
  • 초점의 시각화, 키보드 이동
    1. 키보드 초점이동(포커스링)을 방해하는 자바스크립트 .blur() 를 사용하면 안됩니다.
    2. img 나 span 등 초점을 받을 수 없는 요소에 onclick, onmouseover 등으로 기능을 구현하면 안됩니다. a, input 등의 태그 이용
    3. tabindex 는 정상적인 키보드 이동을 방해하므로 사용하면 안됩니다.
        단, 스크롤이 있는 div 영역 등에는 tabindex="1" 값을 주어 키보드로 스크롤 기능을 이용할 수 있도록 해야 합니다.
    4. onmouseover 등의 이벤트 핸들러만 사용하면 키보드 이용이 불가능하므로 onfocus 등 키보드 이벤트 핸들러를 함께 사용해줍니다.
        onclick 은 마우스와 키보드가 동시에 적용되는 이벤트 핸들러입니다.
    5. ajax 사용에 의한 초점 분실이 생기지 않도록 주의해야합니다.
    6. 레이어 팝업사용시 논리적인 키보드 이동이 가능하도록 컨텐츠를 배치하셔야 합니다.
  • 입력양식
    1. input 타입 image, hidden, submit, button, reset을 제외한 모든 입력양식에는 1대1로 대응하는 label이 존재해야합니다.
    2. label for 값과 id값을 일치시켜주세요. label을 사용하지 못할경우 title 을 사용합니다.
        개발 과정에서 입력양식의 id가 변경된 경우 이와 연결되는 label의 for 속성값도 변경해주세요.
    3. select, textarea 도 포함
    4. select 의 첫번째 option 요소를 title처럼 쓰면 안됩니다.
        예) "성별선택" -> "선택안함" 등의 문구로 변경하여야 합니다.
  • 사용자 요구에 따른 실행
    1. 페이지 로딩시 입력양식으로의 자동 포커스, 입력양식에 값을 입력했을 경우 또는 선택(option, radio, checkbox 등)만으로 새창이 열리거나 포커스가 이동하면 안됩니다.
    2. 오류입력 정정방법의 제공은 제외
    3. 새창으로 열리는 링크에는 title="새창"으로 새창이 열림을 알려야합니다. 특히 window.open등 스크립트로 여는 팝업에는 필수로 넣어야합니다.
  • 오류입력 정정방법의 제공
    모든 입력양식에는 오류입력 정정방법을 제공해야합니다. 입력을 잘못 했을 경우 해당 내용을 고지(alert)하고 해당 위치로 포커스를 이동시켜주어야합니다.
  • 제목 제공
    1. 웹페이지 제목은 문서의 내용을 짧게 요약하여 제공해야하며 서로 배타적이어야 합니다.
        [ 웹페이지 제목에 포함되어야 하는 내용 ]
            - 메뉴 구조상 최하위 단계의 메뉴명
            - 페이지에 탭메뉴가 존재하고, 탭메뉴별로 별도의 페이지가 존재하는 경우 현재 선택된 탭메뉴의 제목
            - 회원가입과 같이 단계가 존재하는 화면의 경우 현재 단계
            - 게시판의 경우 목록 / 상세 / 글쓰기 등 현재화면의 상태
            - 게시판의 상세보기 페이지의 경우 게시글의 제목
    2. iframe 에는 title 로 내부 문서에 대한 짧은 요약 제목을 달아주시고 빈 프레임일 경우 title="내용없음"으로 넣으면 됩니다.
  • 표의 구성
    1. table 에서 th 구조나 문구가 변경된 경우 표의 구성을 간략하게 적어주는 summary 속성의 값도 함께 변경해 주어야 합니다.
    2. 게시판과 같이 개발 로직에 의해 생성 되는 테이블에도 summary / caption 값을 적절하게 넣어 주어야 합니다
        예) summary="게시물을 번호,제목,등록일 순으로 나타낸 표"   /   <caption>공지사항 목록</caption>
        예) summary="게시물을 번호,제목,작성자,등록일 순으로 나타낸 표"   /   <caption>자유게시판 목록</caption>
  • 마크업 오류 방지
    1. 한 페이지에 id 값이 중복 사용되면 안됩니다.
    2. 태그를 열고 닫을 때 태그중첩오류가 발생되면 안됩니다.
  • 기타 유의사항
    1. iframe으로 불러오는 문서도 웹접근성에 맞아야합니다. 내부 문서에는 기본언어표시 lang="ko" xml:lang="ko" 를 넣어주세요
    2. ajax 사용시 초점을 잃지 않게 해야하며 실시간으로 변하는 컨텐츠의 경우 기능버튼에 title 속성으로 다음 데이터가 변경됨을 고지해야합니다.
    3. 레이어팝업 사용시에 순차적으로 키보드 이용이 가능하도록 컨텐츠의 논리적인 구조에 위치해야합니다.
  • 웹접근성 테스트 방법
    1. OpenWAX(Firefox): https://addons.mozilla.org/ko/firefox/addon/openwax
    2. OpenWAX(Chrome): https://chrome.google.com/webstore/detail/openwax/bfahpbmaknaeohgdklfbobogpdngngoe
